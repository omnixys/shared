# 🧾 Changelog

All notable changes in this project will be documented in this file.


## 1.0.0 (2026-04-25)

### ⚠ BREAKING CHANGE

* **Shared:** Complete consolidation of all shared utilities, types, and cross-cutting abstractions.
Legacy helpers, duplicated types, and inconsistent utilities have been removed or restructured.

✨ Features:
- Centralized shared types (DTOs, primitives, utility types) across all Omnixys packages
- Strongly typed helper utilities (object, array, string, date, async, guards)
- Standardized error model with consistent structure and type safety
- Reusable constants, enums, and domain-agnostic abstractions
- Built-in validation helpers and type-safe transformations
- Utility functions for safe serialization (e.g. BigInt, JSON handling)
- Common interfaces for extensibility (factories, adapters, providers)

⚙️ Improvements:
- Eliminated duplication across packages (@omnixys/kafka, @omnixys/security, @omnixys/graphql)
- Strict TypeScript enforcement (no implicit any, improved generics)
- Improved tree-shaking and modular imports
- Consistent naming conventions across all utilities
- Better DX through predictable and discoverable APIs

🧱 Architecture:
- Domain-agnostic, framework-independent design
- Clear separation between types, utils, and abstractions
- Designed as the single source of truth for cross-package logic
- Lightweight and dependency-minimal

🛑 Removed / Changed:
- Removed duplicated helper implementations across services
- Replaced loosely typed utilities with strict generic-based APIs
- Deprecated legacy shared modules and unsafe helpers

📦 Compatibility:
- Requires Node.js >= 20
- Fully compatible with all Omnixys core packages
- Designed for distributed microservice architectures

📚 Notes:
This release establishes @omnixys/shared as the foundational layer for all Omnixys packages,
ensuring consistency, type safety, and maintainability across the entire platform.

## 1.0.0 (2026-04-16)

### ⚠ BREAKING CHANGE

* **Shared:** Complete consolidation of all shared utilities, types, and cross-cutting abstractions.
Legacy helpers, duplicated types, and inconsistent utilities have been removed or restructured.

✨ Features:
- Centralized shared types (DTOs, primitives, utility types) across all Omnixys packages
- Strongly typed helper utilities (object, array, string, date, async, guards)
- Standardized error model with consistent structure and type safety
- Reusable constants, enums, and domain-agnostic abstractions
- Built-in validation helpers and type-safe transformations
- Utility functions for safe serialization (e.g. BigInt, JSON handling)
- Common interfaces for extensibility (factories, adapters, providers)

⚙️ Improvements:
- Eliminated duplication across packages (@omnixys/kafka, @omnixys/security, @omnixys/graphql)
- Strict TypeScript enforcement (no implicit any, improved generics)
- Improved tree-shaking and modular imports
- Consistent naming conventions across all utilities
- Better DX through predictable and discoverable APIs

🧱 Architecture:
- Domain-agnostic, framework-independent design
- Clear separation between types, utils, and abstractions
- Designed as the single source of truth for cross-package logic
- Lightweight and dependency-minimal

🛑 Removed / Changed:
- Removed duplicated helper implementations across services
- Replaced loosely typed utilities with strict generic-based APIs
- Deprecated legacy shared modules and unsafe helpers

📦 Compatibility:
- Requires Node.js >= 20
- Fully compatible with all Omnixys core packages
- Designed for distributed microservice architectures

📚 Notes:
This release establishes @omnixys/shared as the foundational layer for all Omnixys packages,
ensuring consistency, type safety, and maintainability across the entire platform.

## 1.0.0 (2026-04-15)

### ⚠ BREAKING CHANGE

* **Shared:** Complete consolidation of all shared utilities, types, and cross-cutting abstractions.
Legacy helpers, duplicated types, and inconsistent utilities have been removed or restructured.

✨ Features:
- Centralized shared types (DTOs, primitives, utility types) across all Omnixys packages
- Strongly typed helper utilities (object, array, string, date, async, guards)
- Standardized error model with consistent structure and type safety
- Reusable constants, enums, and domain-agnostic abstractions
- Built-in validation helpers and type-safe transformations
- Utility functions for safe serialization (e.g. BigInt, JSON handling)
- Common interfaces for extensibility (factories, adapters, providers)

⚙️ Improvements:
- Eliminated duplication across packages (@omnixys/kafka, @omnixys/security, @omnixys/graphql)
- Strict TypeScript enforcement (no implicit any, improved generics)
- Improved tree-shaking and modular imports
- Consistent naming conventions across all utilities
- Better DX through predictable and discoverable APIs

🧱 Architecture:
- Domain-agnostic, framework-independent design
- Clear separation between types, utils, and abstractions
- Designed as the single source of truth for cross-package logic
- Lightweight and dependency-minimal

🛑 Removed / Changed:
- Removed duplicated helper implementations across services
- Replaced loosely typed utilities with strict generic-based APIs
- Deprecated legacy shared modules and unsafe helpers

📦 Compatibility:
- Requires Node.js >= 20
- Fully compatible with all Omnixys core packages
- Designed for distributed microservice architectures

📚 Notes:
This release establishes @omnixys/shared as the foundational layer for all Omnixys packages,
ensuring consistency, type safety, and maintainability across the entire platform.

## 1.0.0 (2026-04-15)

### ⚠ BREAKING CHANGE

* **Shared:** Complete consolidation of all shared utilities, types, and cross-cutting abstractions.
Legacy helpers, duplicated types, and inconsistent utilities have been removed or restructured.

✨ Features:
- Centralized shared types (DTOs, primitives, utility types) across all Omnixys packages
- Strongly typed helper utilities (object, array, string, date, async, guards)
- Standardized error model with consistent structure and type safety
- Reusable constants, enums, and domain-agnostic abstractions
- Built-in validation helpers and type-safe transformations
- Utility functions for safe serialization (e.g. BigInt, JSON handling)
- Common interfaces for extensibility (factories, adapters, providers)

⚙️ Improvements:
- Eliminated duplication across packages (@omnixys/kafka, @omnixys/security, @omnixys/graphql)
- Strict TypeScript enforcement (no implicit any, improved generics)
- Improved tree-shaking and modular imports
- Consistent naming conventions across all utilities
- Better DX through predictable and discoverable APIs

🧱 Architecture:
- Domain-agnostic, framework-independent design
- Clear separation between types, utils, and abstractions
- Designed as the single source of truth for cross-package logic
- Lightweight and dependency-minimal

🛑 Removed / Changed:
- Removed duplicated helper implementations across services
- Replaced loosely typed utilities with strict generic-based APIs
- Deprecated legacy shared modules and unsafe helpers

📦 Compatibility:
- Requires Node.js >= 20
- Fully compatible with all Omnixys core packages
- Designed for distributed microservice architectures

📚 Notes:
This release establishes @omnixys/shared as the foundational layer for all Omnixys packages,
ensuring consistency, type safety, and maintainability across the entire platform.
